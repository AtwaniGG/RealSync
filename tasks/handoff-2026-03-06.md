# RealSync Handoff — 2026-03-06

## What Was Done This Session

### 1. Emotion Model Upgrade
- Upgraded from MobileNetV2 (128x128, 64.45%) to **EfficientNet-B2 (224x224, 68.79%)**
- Added Zoom compression augmentation: JPEG artifacts, Gaussian blur, downscale-upscale
- Added Mixup augmentation (alpha=0.2), class-weighted loss, configurable backbone
- Serving code (`emotion_model.py`) auto-detects backbone + input size from checkpoint
- **Training paused** at epoch 6/30 via SIGSTOP. Resume: `kill -CONT 78584`
- Backup: `src/models/emotion_weights_mobilenetv2_backup.pth`

### 2. Inference Speed: 7000ms → 150-770ms per frame
- **MPS (Apple Silicon GPU)**: deepfake_model.py + emotion_model.py use `torch.backends.mps`
- **Parallel execution**: inference.py runs deepfake, emotion, identity concurrently via ThreadPoolExecutor(3)
- **Frame semaphore**: app.py processes 1 frame at a time, returns 429 when busy (prevents pile-up)
- **Backend 429 handling**: aiClient.js returns null (skip) instead of mock on 429

### 3. Silent Mock Fallback Fix
- Backend broadcasts "AI Service Offline" alert via WebSocket when AI unreachable
- Frontend shows red warning banner when `source !== 'external'`
- Source label turns red when simulated
- Stub bot reports `captions: false, audio: false` (was lying before)
- Mock response model field includes "(mock)" suffix for visibility

### 4. Bot Fixes
- `BOT_HEADLESS=false` via env var toggle for audio capture
- Join timeout: 90s → 150s (Puppeteer+Zoom takes ~93s)
- Race condition fixed: `bot.cancelled = true` on timeout prevents dual-bot
- Leave flow improved: faster cleanup, browser.close() as fallback

### 5. Config & Alignment
- `AI_TIMEOUT_MS`: 5000 → 30000
- `TEXT_INFERENCE_TIMEOUT`: 5 → 15
- `vite.config.ts` port: 3000 → 5173 (matches backend ALLOWED_ORIGIN)
- DeBERTa model pre-cached (MoritzLaurer/deberta-v3-base-zeroshot-v2.0)
- Unused `emotionScores` variable removed from DashboardScreen.tsx
