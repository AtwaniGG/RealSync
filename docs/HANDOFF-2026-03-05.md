# RealSync Handoff — March 5, 2026

## What Changed This Session

### 1. Deepfake Calibration (biggest change)
**File**: `RealSync-AI-Prototype/serve/deepfake_model.py` ~line 125

**Problem**: The EfficientNet-B4 deepfake model was trained on clean images, but the Zoom bot captures double-compressed frames (Zoom codec + JPEG screenshot). Real faces through Zoom scored 0-14% authenticity — completely broken for demos.

**Fix**: Replaced the simple power-curve calibration with a sigmoid rescale tuned to the actual Zoom output range:
- Raw model output for real faces through Zoom: `raw_authenticity` = 0.005–0.12
- Sigmoid centered at 0.03, steepness 40, scaled to [0.55, 0.95]
- Real faces now score **65–95%** authenticity
- Debug print statement left in: `[deepfake] raw_prediction=... raw_auth=... calibrated=...`

**Important**: This calibration is specifically tuned for Zoom-compressed video. If you switch to a different video source (e.g., raw webcam), the scores will be different. The debug print helps verify.

## Known Issues / What's Left

1. **Emotion model is noisy on compressed video** — it frequently shows Fear/Angry/Sad at low confidence for a normal face through Zoom. We did NOT add suppression (user wanted raw per-frame output). If this is a problem during demo, consider adding a confidence floor (e.g., show "Neutral" if top confidence < 40%).

2. **Bot status can get stuck on "joining"** — if the AI service restarts while a session is active, the backend loses the connection and frames stop flowing. Fix: restart all services together (`bash start.sh`), don't restart individual services mid-session.

3. **Debug print in deepfake_model.py** — there's a `print(f"[deepfake] raw_prediction=...")` on every frame. Remove it before production or it'll flood logs. It's useful for calibration tuning.

4. **The `import math` is inside the function** — works fine but could be moved to the top of the file for cleanliness.

5. **Calibration is Zoom-specific** — the sigmoid parameters (center=0.03, steepness=40, range [0.55, 0.95]) are tuned for Zoom's specific compression artifacts. A different video source would need different tuning. The debug print helps with this.

## How to Verify Calibration

1. Start all services, create session, start bot with a Zoom meeting link
2. Have a real face on camera in Zoom
3. Check AI service logs: `grep "\[deepfake\] raw" /tmp/ai_service.log`
4. Real face should show: `raw_auth` = 0.005–0.15, `calibrated` = 0.65–0.95
5. Dashboard should show: Trust Score 70-90%, Authenticity 65-95%, Risk: low/medium
6. No critical "Visual Manipulation Detected" alerts for real faces

## Architecture Reminder

```
Zoom Bot (Puppeteer, JPEG screenshots every 2s)
  → Backend (Node/Express :4000) POST /api/analyze/frame
    → AI Service (FastAPI :5100) — deepfake + emotion + identity + temporal
      → Response back to backend
        → WebSocket broadcast to dashboard
          → React dashboard (:5173) displays metrics
```

Key files for the pipeline:
- `realsync-backend/bot/ZoomBotAdapter.js` — frame capture (line ~1165)
- `realsync-backend/index.js` — `handleFrame()` (line ~856), `processIngestMessage()` (line ~1200)
- `realsync-backend/lib/aiClient.js` — `analyzeFrame()` HTTP call to AI service
- `RealSync-AI-Prototype/serve/app.py` — `/api/analyze/frame` endpoint
- `Front-End/src/contexts/WebSocketContext.tsx` — WS connection
- `Front-End/src/components/screens/DashboardScreen.tsx` — dashboard display
